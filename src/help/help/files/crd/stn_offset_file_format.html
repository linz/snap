SNAP station offset file format

A station offset file defines the history of station offsets for selected stations as a time series. The offsets of each stations may be either relative to the deformation model used in the adjustment, or to the datum on which it is based. In the latter case the deformation model is ignored of the station. More than one offset can be defined for each station - all the offsets that apply at a given date are added together.

Each offset is specified using the format:

code [coordtype] [offset_type] [mode] type data

The coordtype keyword can be one of:

  • "enu" - the default value, offsets are specified as east/north/up components
  • "height" - the offset is just a vertical component
  • "xyz" - the offset is specified in terms of geocentric XYZ coordinates

The offset_type keyword can be:

  • "offset" - the default value, offsets are relative to the deformation model
  • "deformation" - the offset replaces the deformation model for the station.

The mode keyword can be:

  • "step" - the default value, the data will consist of a date, followed by the coordinate offset. The offset will apply to all station coordinates after that date.`
  • "time_series" - the data consists of a series of values offset date offset ... date offset. The dates must be in sequence. The first offset applies before the first date, and the last offset after the last date. Between two dates the offset is interpolated linearly between the values after those dates.
  • "velocity" - the model data consists of a date and velocity components. The offset at any date is given by applying the velocity relative to the specified date.
Dates are entered using the format "YYYYMMDD" or "dd-mmm-yyyy". All offsets are specified in metres, and velocities in metres per year.
