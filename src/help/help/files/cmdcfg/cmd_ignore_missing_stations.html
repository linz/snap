The ignore_missing_stations command

ignore_missing_stations [yes|no] [report_all|report_none|report_unspecified] [code code ...]

This command defines how SNAP handles missing station codes - codes used in the data files that are not included in the coordinate file.

By default if the data files include stations not in the coordinate file then snap will report on each use of missing codes in the data files and stop with an error when they have all been read.

The ignore_missing_stations command can be used to change this behaviour so that snap will ignore the observations to missing stations and continue to run the adjustment.

The dat2site program can be used to calculate trial coordinates for missing stations.

The ignore_missing_stations command can include the following options:

  • "yes" or "no" to enable or disable ignoring all missing codes
  • "code ... " to list explicit codes that will be ignored.
  • "report_none", "report_unlisted" or "report_all" to choose what is reported in the error listing. The "report_unlisted" option causes does no report listed codes, but does list ignored codes. This is the default. If "report_all"option reports all ignored codes. The "report_none" option does not report ignored codes.
